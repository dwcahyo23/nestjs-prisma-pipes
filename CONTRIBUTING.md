# Contributing to NestJS Prisma Pipes

Contributions are welcome! Please feel free to submit a Pull Request.

## Before You Start

Please make sure you have read and understood the following:

* [Code of Conduct](https://github.com/dwcahyo/nestjs-prisma-pipes/blob/master/CODE_OF_CONDUCT.md)
* [NestJS Prisma Pipes Documentation](https://github.com/dwcahyo/nestjs-prisma-pipes/blob/master/README.md)

## How to Contribute

1. Fork the repository
2. Create your feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add some amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Contribution Guidelines

Please follow these guidelines when contributing to the project:

* Make sure your code is of high quality
* Test your code thoroughly
* Write unit tests if applicable
* Update the documentation if necessary
* Keep your Pull Request focused on a single feature or bug fix
* Do not make changes to the version number or changelog

## Code Style

Please follow the code style of the project:

* Use 2 spaces for indentation
* Use single quotes for strings
* Use TypeScript type annotations where applicable
* Use JSDoc style comments for documenting code

## Communication

Please communicate with the project maintainers through GitHub Issues or Pull Requests.

## Pull Request Checklist

Before submitting a Pull Request, please make sure you have checked the following:

* [ ] The code is of high quality
* [ ] The code is tested thoroughly
* [ ] The documentation is updated if necessary
* [ ] The Pull Request is focused on a single feature or bug fix
* [ ] The code does not change the version number or changelog
